## Docker Configuration

### Image

```yaml
# docker-compose.yml (generated by deploy/playbook_generator.py)
services:
  odoo:
    image: odoo:17.0
    depends_on:
      - db
    environment:
      - HOST=db
      - USER=odoo
      - PASSWORD=${ODOO_DB_PASSWORD}
    volumes:
      - odoo-data:/var/lib/odoo
      - ./config:/etc/odoo
      - ./addons:/mnt/extra-addons
    restart: unless-stopped

  db:
    image: postgres:15
    environment:
      - POSTGRES_DB=postgres
      - POSTGRES_USER=odoo
      - POSTGRES_PASSWORD=${ODOO_DB_PASSWORD}
    volumes:
      - db-data:/var/lib/postgresql/data
    restart: unless-stopped

volumes:
  odoo-data:
  db-data:
```

### Odoo Configuration File

```ini
# /etc/odoo/odoo.conf (mounted into container)
[options]
db_host = db
db_port = 5432
db_user = odoo
db_password = ${ODOO_DB_PASSWORD}
db_name = ${CLIENT_SLUG}_prod

# Security — disable master password after initial setup
admin_passwd = ${ODOO_MASTER_PASSWORD}
list_db = False

# Performance
workers = 2
max_cron_threads = 1
limit_time_cpu = 60
limit_time_real = 120
limit_memory_soft = 640000000
limit_memory_hard = 760000000

# Logging
log_level = warn
logfile = /var/log/odoo/odoo.log
```

## Nginx Configuration

```nginx
# /etc/nginx/sites-available/{client-slug}.hutabyte.id
server {
    listen 80;
    server_name {client-slug}.hutabyte.id;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name {client-slug}.hutabyte.id;

    ssl_certificate     /etc/letsencrypt/live/{client-slug}.hutabyte.id/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/{client-slug}.hutabyte.id/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8069;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Odoo long-polling
        proxy_read_timeout 720s;
        proxy_connect_timeout 720s;
        proxy_send_timeout 720s;
    }

    location /longpolling {
        proxy_pass http://127.0.0.1:8072;
    }
}
```

## Odoo API Access (for Support Bot and AIOps)

The support bot and AIOps components access Odoo via XML-RPC (read-only).

```python
# Connection details stored in age-encrypted client config
# configs/clients/{client-slug}.enc.yaml
{
  "erp_type": "odoo",
  "host": "https://{client-slug}.hutabyte.id",
  "database": "{client-slug}_prod",
  "api_key": "...",        # Odoo API key (read-only user)
  "api_secret": "...",     # Odoo API secret
  "erp_type_version": "17.0",
  "monitoring_id": "uptimerobot-{monitor-id}"
}
```

**Odoo read-only API user setup:**
1. Create a dedicated Odoo user: `hutabyte-bot@internal`
2. Assign only the `base` group (no accounting, HR, or purchase access)
3. Generate an API key for this user (Settings → Users → API Keys)
4. This user can only query `ir.module.module`, `ir.logging`, `ir.cron`
   — all PII-containing models are inaccessible by design
